## 2. 우선순위별 개선 제안

### 🔴 높은 우선순위 — 즉시 개선 권장

| # | 문제 | 현황 | 개선안 |
|---|------|------|--------|
| 1 | **비밀번호 표시/숨김 토글 없음** | Login, Register, Settings 모두 비밀번호 가림 상태 고정 | 👁️ 아이콘 토글 버튼 추가 |
| 2 | **면접 경과 시간 타이머 없음** | 면접 중 시간 감각 상실 가능 | 면접 시작 후 경과 시간 카운터 표시 |

### 🟡 중간 우선순위 — 사용성 향상

| # | 문제 | 현황 | 개선안 |
|---|------|------|--------|
| 7 | **검색 디바운싱 없음** | Jobs/Recruiter 페이지에서 키 입력마다 즉시 필터 → 대량 데이터 시 성능 저하 | 300ms 디바운스 적용 |
| 8 | **설정 변경 이탈 경고 없음** | Settings에서 수정 후 저장 없이 페이지 이동 시 데이터 손실 | `beforeunload` 이벤트 + 확인 모달 |
| 9 | **이력서 업로드 진행률 없음** | 큰 PDF 업로드 시 스피너만 표시 → 진행 상태 불명 | 프로그레스 바 (% 표시) |
| 10 | **면접 기록 페이지네이션 없음** | Dashboard/Profile의 면접 기록이 전부 한 번에 렌더링 | "더 보기" 버튼 또는 무한 스크롤 |
| 11 | **공고 정렬 기능 없음** | 최신순 고정, 사용자가 마감일순/회사명순 등 선택 불가 | 정렬 드롭다운 추가 |
| 12 | **프로필 통계 카드 모바일 대응** | `grid-cols-3` 고정 → 모바일에서 너무 좁음 | `grid-cols-1 sm:grid-cols-3` |

| # | 문제 | 현황 | 개선안 |

| 14 | **비밀번호 강도 표시기** | Register, Settings에서 강도 피드백 없음 | 색상 게이지 (약함/보통/강함) |
| 15 | **음성 파형 시각화** | 면접 중 음성 입력 피드백이 텍스트만 | 실시간 Waveform 시각화 |
| 16 | **스크롤 애니메이션** | 랜딩 페이지 기능 카드가 즉시 표시 | Intersection Observer 기반 fade-in |
| 17 | **다크/라이트 모드 전환** | 다크 모드 전용 | 시스템 설정 연동 + 수동 토글 |

## 3. 페이지별 상세 분석

### 3-1. 랜딩 페이지 (`page.tsx`)

**UI 요소**: Sticky 헤더, 히어로 섹션 (CTA), 5개 기능 카드, 모달 3종 (Login/Register/ForgotPassword)

**사용자 피드백**:
- ✅ 로그인 여부에 따라 CTA 텍스트 동적 변경
- ✅ 모달 전환 상태 관리

**문제점**:
- 페이지 로딩 시 인증 상태 확인 중 빈 화면 → 스켈레톤 UI 필요
- 스크롤 기반 노출 애니메이션 미적용
- 푸터 없음 (서비스 정보, 이용약관)
- SEO: Open Graph 태그 미적용

---

### 3-2. 대시보드 (`dashboard/page.tsx`)

**UI 요소**: 환영 배너, 이력서 관리 (드래그앤드롭), 환경 테스트 (카메라/마이크), 면접 시작 CTA, 면접 기록

**사용자 피드백**:
- ✅ 업로드 상태 스피너, 디바이스 체크 아이콘
- ❌ `window.confirm()` 사용 → 커스텀 모달 필요
- ❌ 이력서 업로드 진행률 표시 없음

**문제점**:
- 드래그앤드롭 영역에 `role="button"`, `aria-label` 없음
- 마이크 레벨 바에 `role="progressbar"` 없음
- 면접 기록 대량 시 페이지네이션 없음

---

### 3-3. 면접 페이지 (`interview/page.tsx`)

**UI 요소**: Setup(카메라 프리뷰+시작), Interview(상태 바/진행 바/채팅/STT/미디어 컨트롤), Coding(코딩 테스트), Whiteboard(설계), Report(차트/PDF)

**사용자 피드백**:
- ✅ 상태별 색상 배지 (ready/listening/speaking/processing)
- ✅ 진행 바 시각적 구분
- ✅ `EventToastContainer` 실시간 알림
- ❌ `alert()` 사용 (카메라 실패, 면접 시작 실패)
- ❌ 면접 경과 시간 타이머 없음

**문제점**:
- 미디어 컨트롤 버튼에 `aria-label` 없음
- 채팅 메시지 영역에 `role="log"`, `aria-live="polite"` 없음
- 코딩/화이트보드 Phase "건너뛰기" 시 확인 다이얼로그 없음
- 네트워크 끊김 시 재연결 UI 피드백 부재

---

### 3-4. 공고 페이지 (`jobs/page.tsx`)

**UI 요소**: 검색바 + 필터(직무/경력), 공고 카드 리스트 (아코디언 상세), 인사담당자: CRUD 버튼, 지원자: "이 공고로 면접 연습하기" 버튼, 등록/수정/삭제 모달

**사용자 피드백**:
- ✅ 로딩 스피너, 빈 리스트 상태 UI
- ✅ 폼 유효성 검증 에러 메시지
- ❌ CRUD 성공 후 토스트 알림 없음
- ❌ 검색 디바운싱 없음

**문제점**:
- 아코디언에 `aria-expanded`, `aria-controls` 없음
- 모달에 `role="dialog"` 없음
- 공고 정렬 기능/페이지네이션 없음

---

### 3-5. 채용담당자 대시보드 (`recruiter/page.tsx`)

**UI 요소**: 환영 배너 + CTA, 통계 카드 4종, 검색 + 상태 필터 탭, 공고 CRUD, 모달 2종

**사용자 피드백**:
- ✅ 빈 상태 UI ("첫 공고 등록하기" CTA)
- ✅ 모달 overlay 드래그 오작동 방지 구현
- ❌ CRUD 성공 후 피드백 없음

**문제점**:
- Jobs 페이지와 대량 코드 중복 (공고 CRUD 로직, 모달 폼)
- 지원자 수 통계 데이터 연동 없음
- 검색 입력에 `aria-label` 없음

---

### 3-6. 프로필 페이지 (`profile/page.tsx`)

**UI 요소**: 프로필 헤더 (아바타 이니셜, 역할 뱃지), 개인정보 읽기 전용, 면접 통계 3종, 면접 기록 리스트, 회원탈퇴 모달

**사용자 피드백**:
- ✅ 점수 기반 색상 피드백 (80+/60+/40+/그 이하)
- ✅ 빈 상태 "첫 면접 시작하기" CTA

**문제점**:
- 통계 카드 `grid-cols-3` 고정 → 모바일에서 좁음
- 면접 기록 페이지네이션 없음
- 프로필 이미지 업로드 기능 없음
- 면접 기록 상세가 `window.open()` → SPA 내 라우팅 권장

---

### 3-7. 설정 페이지 (`settings/page.tsx`)

**UI 요소**: 기본 정보 수정 폼 (이메일 읽기전용), 비밀번호 변경 폼

**사용자 피드백**:
- ✅ 성공/에러 인라인 메시지
- ✅ `disabled` 상태 (필수 미입력 시)

**문제점**:
- `<label>` ↔ `<input>` `htmlFor`/`id` 연결 없음
- 성별/회원 유형 버튼에 `role="radio"` 없음
- 비밀번호 강도 표시기 없음
- 변경사항 미저장 이탈 경고 없음
- 전화번호 형식 검증 없음
- 회원 유형 변경 시 확인 다이얼로그 없음

---

### 3-8. 공통 헤더 (`Header.tsx`)

**UI 요소**: 로고, 역할별 네비게이션, 역할 뱃지, 로그아웃 버튼

**문제점**:
- `<nav>` 시맨틱 태그 미사용
- 현재 활성 페이지 표시 없음 (`aria-current="page"`)
- **모바일 반응형 메뉴 전혀 없음** → 작은 화면에서 겹침 발생
- 알림/프로필 드롭다운 없음
- 스킵 네비게이션 링크 없음

---

### 3-9. 공통 모달 (`Modal.tsx`)

**UI 요소**: Overlay + Glass-card 본체 + 제목/닫기(X) + children slot

**양호한 점**:
- ✅ Overlay 드래그 오작동 방지 (`mouseDownTarget`)
- ✅ `maxHeight: 90vh` + `overflowY: auto`

**문제점**:
- `role="dialog"`, `aria-modal="true"`, `aria-labelledby` 없음
- 포커스 트랩 미구현 (Tab 키가 모달 밖으로 나감)
- Escape 키로 닫기 미구현
- `body` 스크롤 잠금 미구현
- 열림/닫힘 애니메이션 exit 부재

---

### 3-10. 로그인 모달 (`LoginModal.tsx`)

**UI 요소**: 이메일/비밀번호, 소셜 로그인 3종 (카카오/Google/네이버), 비밀번호 찾기/회원가입 전환

**양호한 점**:
- ✅ Enter 키 제출 (`handleKeyDown`)
- ✅ 에러 메시지 표시

**문제점**:
- 비밀번호 표시/숨김 토글 없음
- `<label>` ↔ `<input>` 연결 없음
- "로그인 상태 유지" 체크박스 없음
- 소셜 로그인 아이콘(SVG) 미사용

---

### 3-11. 회원가입 모달 (`RegisterModal.tsx`)

**UI 요소**: 이메일 (중복 확인), 비밀번호/확인, 이름, 회원 유형 토글, 선택 입력 (생년월일/성별/주소/전화번호)

**양호한 점**:
- ✅ 이메일 중복 실시간 확인 (`onBlur`)
- ✅ 필수 필드 `*` 표시

**문제점**:
- 비밀번호 강도 표시기 없음
- **이용약관/개인정보처리방침 동의 체크박스 없음 (법적 필수사항)**
- 성공 시 `alert()` 사용 → 인라인 성공 메시지 또는 자동 로그인 권장
- 전화번호 형식 마스킹/검증 없음

---

### 3-12. 전역 스타일 (`globals.css`)

**구성**: CSS 변수 체계, `.gradient-text`, `.btn-gradient`, `.glass-card`, `.modal-overlay`, `.input-field`, 커스텀 스크롤바, 애니메이션

**문제점**:
- `prefers-reduced-motion` 미디어 쿼리 없음 → 모션 민감 사용자 미고려
- `prefers-color-scheme` 미지원 (다크 모드 전용)
- `:focus-visible` 스타일 미정의 → 키보드 포커스 링 불명확
- `--glass-border` 변수 사용되지만 정의되지 않음
- 반응형 타이포그래피 (`clamp()`) 미사용

---

### 3-13. 루트 레이아웃 (`layout.tsx`)

**양호한 점**:
- ✅ `<html lang="ko">` 한국어 접근성
- ✅ `AuthProvider` 전역 감싸기

**문제점**:
- 글로벌 에러 바운더리 (`error.tsx`) 없음
- 글로벌 로딩 UI (`loading.tsx`) 없음

## 4. 접근성(a11y) 종합

| 문제 | 영향 영역 | 심각도 |
|------|-----------|--------|
| `aria-label`, `role` 속성 전반 누락 | 모든 페이지 | 🔴 높음 |
| 모달 포커스 트랩 / Escape 닫기 미구현 | Modal, LoginModal, RegisterModal | 🔴 높음 |
| `<label>` ↔ `<input>` 연결 (`htmlFor`) 없음 | 모든 폼 | 🔴 높음 |
| `<nav>` 시맨틱 태그 미사용 | Header | 🟡 중간 |
| `prefers-reduced-motion` 미지원 | globals.css | 🟡 중간 |
| `:focus-visible` 스타일 미정의 | globals.css | 🔴 높음 |
| Skip Navigation 링크 없음 | Header/Layout | 🟡 중간 |
