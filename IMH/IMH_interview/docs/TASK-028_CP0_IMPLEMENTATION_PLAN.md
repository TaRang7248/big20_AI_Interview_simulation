# TASK-028 CP0 Implementation Plan: Business Statistics MVP

This document outlines the implementation plan for **TASK-028 CP0**, focusing on the **Business Statistics (Track A)** subset.
This implementation provides the foundational "Query-Only" layer for the Admin Dashboard.

---

## 1. Goal & Scope (CP0)

### Goal
Implement a Read-Only Statistics Layer to provide business insights (Counts, Averages, Rates) derived solely from PostgreSQL snapshots.

### Scope (Track A Only)
- **Package**: `packages/imh_stats`
- **Query Types**:
  - **Type 1 (Real-time)**: Simple counts (e.g., "Current applicants today").
  - **Type 2 (Derived)**: Aggregations with Caching (e.g., "Weekly Average Score").
- **Contracts**:
  - **Source**: PostgreSQL (`interviews`, `evaluations` tables).
  - **Cache**: Redis (Read-Through only, No Write-Back).
  - **Engine**: No dependency on `SessionEngine` or Command Logic.

### Out of Scope
- Track B (Operational Observability) / Logs.
- Type 3 (Multi-dim) / Type 4 (Correlation).
- UI Implementation.

## 2. Architecture & Components

### 2.1 Package Structure
```
packages/imh_stats/
├── __init__.py
├── dtos.py           # StatsRequest, StatsResponse, Meta
├── service.py        # StatisticsQueryService (Facade)
├── repository.py     # StatisticsRepository (PG), RedisStatsRepository (Cache)
├── enums.py          # StatType (COUNT, AVERAGE, RATIO)
```

### 2.2 DTO Design
Standardized response format for all statistics.
```python
@dataclass
class StatsMetaDTO:
    query_type: str        # "REALTIME" | "CACHED"
    period_from: datetime | None
    period_to: datetime | None
    group_by: str | None
    as_of: datetime        # Data snapshot time
    is_cached: bool
    ttl_remaining: int | None

@dataclass
class StatsResponseDTO[T]:
    meta: StatsMetaDTO
    result: T              # The actual statistic value/list
```

### 2.3 Repository Layer
- **`StatisticsRepository`**:
  - Direct SQL queries to PostgreSQL.
  - **Strictly Read-Only** (SELECT only).
  - Uses `interviews` and `evaluations` tables.
  - *No domain entity hydration* (Pure Data Projection).

- **`RedisStatsRepository`**:
  - Key Format: `stats:{category}:{query_hash}`
  - TTL Strategy:
    - Type 1: No Cache (or very short 1m).
    - Type 2: 1h ~ 24h.

### 2.4 Service Layer (`StatisticsService`)
- Orchestrates `Read-Through` caching pattern.
- **Logic**:
  1. Check Redis (if cacheable).
  2. If Hit -> Return Cached Result (with `is_cached=True`).
  3. If Miss -> Query PostgreSQL.
  4. Save to Redis (if cacheable).
  5. Return Result (with `is_cached=False`).

## 3. Implementation Steps

1.  **Setup Package**: Create `packages/imh_stats` and empty files.
2.  **DTO Implementation**: Define `StatsResponseDTO` and Meta.
3.  **PG Repository**: Implement `StatisticsRepository` with methods:
    - `get_daily_applicant_counts(start, end)`
    - `get_score_distribution(job_id)`
    - `get_pass_rate(job_id)`
4.  **Redis Repository**: Implement `RedisStatsRepository` for caching results.
5.  **Service Layer**: Implement `StatisticsService` tying PG and Redis together.
6.  **Dependency Injection**: Register in `IMH/bootstrap.py` (if applicable) or prepare for API integration.

## 4. Verification Plan

### Script: `scripts/verify_task_028_cp0.py`

**Test Cases:**
1.  **Authority Check**:
    - Ensure data matches PostgreSQL rows (Insert dummy data -> Query Stats -> Match).
2.  **Cache Behavior (Read-Through)**:
    - Call Type 2 Stat -> Cache Miss (DB Query).
    - Call again -> Cache Hit (Redis Query).
    - Verify `as_of` does not change on Hit.
3.  **Isolation Check**:
    - Verify `SessionEngine` is NOT imported/called.
    - Verify no INSERT/UPDATE queries are generated by Stats package.
4.  **Contracts**:
    - Assert `Write-Back` to PG is impossible (Code structure check).
    - Assert Response contains mandatory Metadata (`as_of`).

---

## 5. Security & Performance Policies

- **Input Validation**: Date ranges must be sanitized.
- **Max Limit**: Period queries constrained to max 90 days to prevent aggregation overload (CP0 constraint).
- **Index Usage**: Ensure queries hit existing indices on `created_at`, `job_id`, `status`.
