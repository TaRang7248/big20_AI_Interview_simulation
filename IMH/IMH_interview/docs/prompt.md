# PROMPTS
(IMH AI 코딩 에이전트 공식 실행 프롬프트 모음)

본 문서의 프롬프트는 **매번 그대로 복사/붙여넣기**하여 사용한다.  
프롬프트를 임의로 수정하거나 축약해서는 안 된다.

---

## 1. PLAN_ONLY 프롬프트 (허락 전 – 계획만)

너는 IMH 프로젝트의 AI 코딩 에이전트다.

필수 규칙:
- `docs/00_AGENT_PLAYBOOK.md`
- `docs/CURRENT_STATE.md`
- `docs/TASK_QUEUE.md`
를 **반드시 먼저 읽고 시작해라**.

- ACTIVE 상태의 TASK만 수행 가능하다.
- 코드 변경 전에는 반드시 변경 제안서(Plan Only)를 작성한다.
- 사용자 허락 전에는 코드 생성/수정/대규모 diff 출력 금지.
- 에러는 MD가 아니라 `logs/*.log` 로 남긴다.

현재 작업:
- TASK-001 로깅 기반 구축

출력 형식:
1) 변경 제안서
   - 생성/수정 파일
   - 설계 이유
   - 로깅 정책(레벨/파일 분리/로테이션)
   - 영향 범위
   - 롤백 방법
2) 사용자 허락 요청 (한 줄)

---

## 2. IMPLEMENT 프롬프트 (허락 후 – 구현)

사용자가 변경 제안서를 허락했다.

지침:
- 승인된 계획 범위 내에서만 구현한다.
- 공통 로거는 재사용 가능한 형태로 설계한다.
- 모든 예외는 `logger.exception()`으로 기록한다.

구현 후 반드시 수행:
- 로그 파일 생성 여부 검증
- `docs/DEV_LOG.md` 업데이트
  - 변경 요약
  - 테스트 방법
  - 로그 파일 경로

출력 형식:
1) 구현 요약
2) 생성/수정 파일 목록
3) 테스트 방법 (재현 가능한 커맨드)
4) 로그 파일 위치
5) 다음 작업 제안 (최대 2개)

---

## 3. TASK_CLOSE 프롬프트 (작업 종료 정리)

TASK 구현이 완료되었다.

수행 지침:
- `docs/TASK_QUEUE.md`에서 해당 TASK를 DONE으로 이동
- `docs/DEV_LOG.md`에 완료 기록 추가
- 다음 ACTIVE 후보 TASK 제안 (Queue 수정은 사용자 승인 필요)

출력 형식:
1) 완료 요약
2) DONE 처리 제안
3) 다음 TASK 후보 1~2개
