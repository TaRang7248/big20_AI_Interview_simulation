# INTERVIEW_POLICY_SPEC
(IMH AI 면접 프로그램 – 면접 동작 정책 명세서)

본 문서는 공고 기반 AI 면접 시스템의 **동작 정책(Product Behavior Policy)**을 정의한다.  
UI 문서, ERD 문서, TASK 문서와 구분되며, “면접이 실제로 어떻게 동작해야 하는지”를 고정하는 상위 계약 문서이다.

---

# 1. 문서 목적

- 공고 기반 AI 면접의 동작 규칙을 명문화한다.
- 관리자 설정이 면접 엔진에 어떻게 반영되는지 정의한다.
- 실전 모드와 연습 모드의 정책 차이를 명확히 한다.
- 질문 전환 / 종료 / 평가 노출 / 결과 공개 정책을 고정한다.
- 향후 DB 도입 및 RAG 도입 시 기준 문서로 사용한다.

---

# 2. 용어 정의

- **공고(Job Posting)**: 관리자가 등록한 채용 공고. 면접 정책(옵션)과 모델 선택의 단위.
- **세션(Interview Session)**: 특정 지원자와 특정 공고(또는 연습 모드)에서 생성되는 면접 실행 단위.
- **실전 모드**: 공고에 지원한 뒤 진행하는 1차 AI 면접.
- **연습 모드**: 모든 유저에게 제공되는 AI 면접 연습 기능.
- **침묵(Silence)**: 일정 시간 이상 입력(발화/텍스트)이 없는 상태.
- **무응답(No Answer)**: 질문에 대한 실질적 답변이 생성되지 않은 상태로 침묵이 지속된 케이스.
- **근거 데이터(Evidence)**: 평가 점수 산출을 뒷받침하는 내부 기록 및(옵션에 따라) 사용자 노출 요약 근거.

---

# 3. 면접 모드 정의

## 3.1 실전 1차 AI 면접 (공고 기반)

- 카메라 필수
- 마이크 필수
- 일시정지 불가
- 재시도 불가
- 재질문 기본 비활성 (관리자 옵션으로 활성화 가능하나 기본은 비활성)
- 면접 중 점수 비공개(진행도만 표시)
- 면접 종료 후 결과 공개 여부/범위/시점은 관리자 설정에 따름(단, 2주 이내 합/불합 자동 통지 보장)

## 3.2 AI 면접 연습 모드

- 카메라/마이크 필수
- 일시정지 허용
- 재시도 허용
- 재질문 요청 허용
- 상세 피드백 기본 제공(면접 종료 후)

---

# 4. 실전/연습 모드 정책 차이표

| 항목 | 실전(공고 기반 1차) | 연습(모든 유저) |
|---|---|---|
| 카메라/마이크 | 필수 | 필수 |
| 질문 출력 | TTS 기본, 텍스트 표시는 공고 옵션 | TTS 기본, 텍스트 표시 기본 ON(추후 옵션화 가능) |
| 답변완료 버튼 | 사용 | 사용 |
| 침묵 자동완료 | 사용(공고 옵션) | 사용(기본값 사용, 추후 옵션화 가능) |
| 질문당 제한시간 | 사용(공고 옵션) | 사용(기본값 사용, 추후 옵션화 가능) |
| 일시정지 | 불가 | 허용 |
| 재질문 | 기본 비활성(옵션 가능) | 허용 |
| 재시도(답변 재진행) | 불가 | 허용 |
| 점수/피드백 노출 | 면접 중 비공개, 종료 후 공개 범위/시점은 공고 옵션 | 종료 후 상세 피드백 기본 |
| 세션 중단 시 | 중단 종료 처리(중단 표기) | 재시작/재진행 허용(사용자 선택) |

---

# 5. 면접 시작 조건

- 카메라 및 마이크 권한 승인 필수
- 권한 미승인 시 면접 시작 불가
- 환경 테스트(권한/장치 확인) 완료 후 면접 시작 가능

---

# 6. 공고 기반 세션 생성 정책

- 실전 모드 세션은 **공고 단위 정책(옵션/모델/가중치/필수 질문/결과 공개 정책)**을 기반으로 생성된다.
- 공고 등록 후 정책 중 “수정 불가 항목”은 변경되지 않는다(예: 모델 선택).
- 세션 생성 시점 기준으로 공고 설정이 적용되며, 세션 시작 이후 공고 설정 변경은 **진행 중 세션에 영향을 주지 않는다**(세션은 고정된 정책으로 동작).

---

# 7. 질문 출력 방식

관리자는 공고 등록 시 다음 옵션을 설정할 수 있다.

- **TTS만 제공**
- **TTS + 텍스트 동시 제공**

텍스트 표시 여부는 면접 태도 평가(집중력/지시 이해 등)에 간접적으로 영향을 줄 수 있다.

---

# 8. 질문 전환 조건

질문은 다음 조건 중 하나를 만족하면 종료된다.

1. 지원자가 **‘답변 완료’ 버튼**을 클릭
2. **침묵**이 관리자 설정 시간(기본 10~15초) 초과
3. **질문당 제한시간**(기본 2분) 초과

침묵 자동완료 시간은 공고별로 설정 가능하며, 질문당 제한시간 역시 공고별로 설정 가능하다.

---

# 9. 침묵(Silence) 처리 정책

침묵은 다음 두 경우로 구분한다.

## 9.1 “답변 후 침묵” 케이스
- 면접자가 이미 답변을 충분히 한 뒤 침묵이 발생한 경우.
- 처리 원칙:
  - 시스템은 “답변이 완료되었다면 답변완료 버튼을 눌러달라”는 안내를 제공한다.
  - 침묵이 설정 시간을 초과하면 질문을 종료하고 다음 질문으로 진행한다.
  - 이 케이스는 **무응답 페널티로 처리하지 않는다**(별도 부정 점수 반영 없음).

## 9.2 “무응답 침묵” 케이스(답변 없이 침묵)
- 고민했으나 답변이 나오지 않는 상황으로 간주한다.
- 처리 원칙:
  - 침묵이 설정 시간을 초과하면 질문을 종료하고 다음 질문으로 진행한다.
  - 이 케이스는 **부정적 점수(또는 무응답 페널티)**로 반영될 수 있다.
  - 무응답 반영 방식은 평가 엔진의 루브릭/정책에 따라 “무응답 플래그”로 전달한다.

---

# 10. 종료 조건

면접 종료는 다음 기준에 의해 결정된다.

- 설정된 **총 질문 수** 도달
- **점수 기반 조기 종료 조건** 만족
- 관리자 지정 **필수 질문** 완료

면접은 시간 총량이 아닌 질문 수 및 점수 기준을 우선한다.

## 10.1 최소 질문 수 보장 정책

- 실전(공고 기반 1차 AI 면접)의 기본 최소 질문 수는 10문항을 권장 기준으로 한다.
- 공고별 설정에 따라 총 질문 수는 변경 가능하나,
  점수 기반 조기 종료는 최소 질문 수(기본 10문항)를 수행한 이후에만 허용된다.
- 최소 질문 수는 답변 완료 기준으로 계산한다.


---

# 11. 평가 정책

- 내부적으로는 **질문 단위 평가**를 수행할 수 있다(성능/속도 최적화 목적).
- 사용자에게 점수/피드백은 **면접 종료 후**에만 노출한다.
- 면접 중 사용자에게 노출되는 정보는 **진행도(Progress)**로 제한한다.

## 11.1 평가 가중치 조정 범위(관리자 설정)
- 관리자는 평가 축별 가중치를 조정할 수 있다.
- 각 축의 가중치 범위는 **10% ~ 90%**를 기본 정책으로 한다.
- 최종 합산은 “가중치 적용 후 집계”를 원칙으로 한다(집계 방식 상세는 평가/리포팅 문서에서 확정).

---

# 12. 결과 공개 정책(관리자 설정)

실전 모드(공고 기반)에서 결과 공개는 **관리자 설정을 우선**한다.

## 12.1 공개 시점 옵션(공고 단위)
1. **즉시 공개**
2. **지정 시각 공개**: `YYYY-MM-DD HH:00:00`
3. **관리자 승인 시 공개**

## 12.2 최소 통지 보장(강제 정책)
- 위 설정과 무관하게, 면접자가 면접을 진행한 시점으로부터 **2주(14일) 이내**에는
  최소한 **합/불합 결과가 자동으로 통지**되도록 보장한다.

## 12.3 공개 범위 옵션(공고 단위)
- 합/불합만 공개
- 요약 공개
- 상세 피드백 공개
- 결과 비공개(단, 2주 이내 합/불합 자동 통지 보장 규칙은 적용)

---

# 13. 질문은행 / RAG Fallback 정책 (6천 Q/A 활용)

- 질문 생성 품질이 낮거나 실패한 경우 질문은행을 활용한다.
- 공고 직무 및 인재상 태그 기반으로 후보를 검색한다.
- 관리자 지정 필수 질문은 우선 적용한다.
- 질문은행은 “시점 고정”이 아니라 “조건 기반 fallback”으로 사용한다.

---

# 14. 관리자 필터/조회 정책(지원자 목록)

관리자(인사담당자)는 공고별 지원자 목록에서 다음 필터를 사용할 수 있어야 한다.

## 14.1 필수 필터(요구사항)
- 날짜(기간)
- 접수 상태(접수/진행/완료/중단 등 상태 기반)
- 합/불합(판정 결과)

## 14.2 추천 필터(추가 제안)
- 점수 구간(예: 상/중/하 또는 0~100 구간)
- 평가 축별 약점 필터(예: 의사소통 낮음, 문제해결 낮음 등)
- 면접 중단 여부(중단 세션만 보기)
- 재지원/중복 지원 여부(가능한 경우)
- 공고 내 포지션/모집분야(공고가 복수 포지션을 가질 때)

## 14.3 면접 세션 상태값 정의 (Status Enum)
면접 세션은 다음 상태값 중 하나를 가진다:

- APPLIED: 지원 완료, 면접 시작 전 상태
- IN_PROGRESS: 면접 진행 중
- COMPLETED: 정상 종료
- INTERRUPTED: 면접 중단 종료
- EVALUATED: 평가 및 결과 확정 완료

이 상태값은 관리자 필터 및 통계의 기준이 된다.

---

# 15. 세션 중단/복구 정책

## 15.1 실전 모드(공고 기반 1차)
- 면접 도중 세션이 중단되면 해당 면접은 **중단 종료** 처리한다.
- 관리자는 지원자 상세 화면에서 해당 세션이 **면접중단**임을 확인할 수 있어야 한다.
- 중단 원인(사용자 종료/시스템 오류) 판별은 정책 범위 밖이며, 본 문서는 “중단 처리/표기”만 고정한다.

## 15.2 연습 모드
- 면접 도중 세션이 중단되면 사용자는 **재진행(재시작/이어하기)**을 선택할 수 있다.

---

# 16. 관리자 설정 항목(공고 등록 시)

공고 등록 시 설정 가능한 정책 항목:

- 질문 표시 방식 (TTS / TTS+텍스트)
- 침묵 자동완료 시간
- 질문당 제한시간
- 총 질문 수
- 점수 기반 종료 여부
- 평가 가중치 조정(10~90% 범위)
- 필수 질문 지정
- 면접관 페르소나 추가 설정
- 결과 공개 범위
- 결과 공개 시점(즉시/지정 시각/관리자 승인)
- 사용할 모델 선택(공고 단위, 등록 후 수정 불가)

---

# 17. 모델 선택 정책

- **실전(공고 기반 1차)**: 관리자가 공고 등록 시 모델을 지정한다(공고 등록 후 수정 불가).
- **연습 모드**: 면접자가 연습 시작 전(환경 체크 단계)에서 사용할 모델을 선택할 수 있다.

---

# 18. 근거 데이터(Evidence) 정책

## 18.1 내부 저장 목적(필수)
- 질문별 점수 산출 근거(요약 지표 및 플래그)
- 침묵 시간 및 무응답 플래그
- 발화 길이(요약)
- 분석 지표 요약(Emotion/Voice/Visual의 핵심 요약값)

## 18.2 사용자 노출 정책(옵션)
- 연습 모드: 상세 제공(기본)
- 실전 모드: 관리자 설정(공개 범위)에 따름
