# 26.01.23(금) 기업별 ‘인재상’ 의 루브릭 평가항목 기술구현

대부분의 AI 면접 서비스가 범용적인 질문만 던지는 것과 달리, **'기업별 인재상'** 데이터가 들어가면 이 시스템은 \*\*"맞춤형 채용 솔루션"\*\*으로 격상됩니다. 기업 인사담당자들이 가장 목말라하는 부분이 바로 \*\*"이 사람이 우리 회사 문화(Culture Fit)와 맞는가?"\*\*이기 때문입니다.

확보하신 인재상 데이터를 루브릭에 녹여내고, 이를 시스템적으로 구현하는 **가장 현실적이고 기술적인 방법**을 제안해 드립니다.

---

### **1\. 인재상 데이터의 루브릭 배치 전략**

인재상은 기존의 '직무 역량(Hard Skill)'이나 '논리력(Logic)'과는 결이 다릅니다. 따라서 기존 항목에 섞기보다는 \*\*별도의 \[가변형 평가 영역\]\*\*으로 설계하는 것이 좋습니다.

저는 이를 \*\*"조직 적합성 (Culture & Value Fit)"\*\*이라는 제 5의 영역으로 신설하고, **Dynamic Rubric(동적 루브릭)** 기술을 적용할 것을 권장합니다.

#### **변경된 루브릭 구조**

| 평가 영역 | 고정/가변 | 세부 항목 | 설명 |
| ----- | ----- | ----- | ----- |
| 직무 역량 | 고정 | 기술 이해도, 코드 품질 | Python, CS 지식 등 |
| 문제 해결력 | 고정 | 논리적 전개, 접근 방식 | 문제 분해 및 해결 과정 |
| 의사소통 | 고정 | 구조화(STAR), 요지 파악 | 말하는 스킬 |
| 태도/비언어 | 고정 | 시선, 목소리, 표정 | Vision/Audio 분석 |
| **✨ 조직 적합성** | **가변 (Dynamic)** | **핵심 가치 일치도** | **선택한 기업의 인재상(도전, 협력, 창의 등) 반영** |

---

### **2\. 구체적인 구현 시나리오 (Implementation Strategy)**

"인재상"은 보통 추상적인 단어(예: 창의, 열정)로 되어 있습니다. 이를 AI가 평가하려면 \*\*'행동 지표(Behavioral Indicator)'\*\*로 변환하여 벡터 DB에 저장하고, RAG로 불러와야 합니다.

#### **Step 1: 데이터 전처리 (Data Processing)**

수집한 기업별 인재상을 단순히 텍스트로 저장하지 말고, \*\*"어떤 행동을 해야 점수를 줄 것인가?"\*\*로 변환하여 저장합니다.

* **원본 데이터:** "창의적인 인재 (삼성전자)"  
* **변환 데이터 (System Prompt용):**  
  * *"지원자가 기존의 방식을 답습하지 않고 새로운 시도를 한 경험을 말했는가?"*  
  * *"실패를 두려워하지 않고 개선하려 했던 구체적 사례가 있는가?"*

#### **Step 2: DB 스키마 설계 (Vector DB & RDB)**

**Table: company\_values (RDB 또는 Vector Meta)**

JSON

{  
  "company\_name": "Naver",  
  "core\_values": \[  
    {  
      "keyword": "사용자 중심 (User Obsession)",  
      "description": "기술보다는 사용자의 불편함을 먼저 고민하는 태도",  
      "evaluation\_prompt": "답변에서 '기술적 난이도'보다 '사용자의 피드백'을 중요하게 언급했는지 체크하라."  
    },  
    {  
      "keyword": "주도적 성장",  
      "description": "시키지 않아도 스스로 학습하고 성장하는 태도",  
      "evaluation\_prompt": "학습 과정에서 자기주도적인(Self-directed) 액션이 있었는지 체크하라."  
    }  
  \]  
}

#### **Step 3: 프롬프트 엔지니어링 (Dynamic Prompt Injection)**

사용자가 면접 시작 전 "네이버"를 선택하면, 백엔드(FastAPI)가 해당 기업의 core\_values를 조회하여 LLM의 시스템 프롬프트에 \*\*동적으로 삽입(Injection)\*\*합니다.

**\[LLM System Prompt Template\]**

Python

system\_prompt \= f"""  
당신은 {target\_company}의 면접관입니다.  
지원자의 답변을 평가할 때, 아래의 \*\*\[기업 핵심 가치\]\*\*를 기준으로 '조직 적합성' 점수(1\~5점)를 매기십시오.

\[기업 핵심 가치\]  
1\. {value\_1\_keyword}: {value\_1\_prompt}  
2\. {value\_2\_keyword}: {value\_2\_prompt}

... (중략) ...

만약 지원자의 답변이 위 핵심 가치와 전혀 관련이 없다면 '조직 적합성' 점수를 낮게 부여하고,  
관련된 경험(STAR)이 언급되었다면 가산점을 부여하십시오.  
"""

#### **Step 4: 평가 로직 (Semantic Similarity Scoring)**

단순히 프롬프트에만 의존하지 말고, \*\*임베딩 유사도(Cosine Similarity)\*\*를 활용하여 객관적인 수치를 뽑을 수 있습니다.

1. **기업 인재상 벡터:** V\_target \= Embedding("끊임없는 도전과 혁신")  
2. **지원자 답변 벡터:** V\_answer \= Embedding("기존 라이브러리의 한계를 느껴 직접 코드를 수정하여 성능을 개선했습니다.")  
3. **유사도 계산:** Score \= CosineSimilarity(V\_target, V\_answer)  
   * Score가 0.8 이상이면 → **조직 적합성 5점** 부여.

### **💡 30년차 개발자의 핵심 조언**

1. **"질문"도 바뀌어야 합니다:** 인재상이 '협력'이라면, AI 면접관이 기술 질문 후에 \*\*"그 프로젝트를 진행하면서 동료와 의견 충돌은 없었나요?"\*\*라는 인성(Soft Skill) 꼬리 질문을 던지도록 LangChain 로직을 짜야 합니다. 이것이 진정한 '맞춤형'입니다.  
2. **보여주기식 기능이 아닙니다:** 이 기능은 B2B(기업 대상) 세일즈 포인트로 가장 강력합니다. "우리 회사 인재상을 넣으면, 우리 회사에 맞는 사람을 AI가 골라줍니다"라고 어필할 수 있는 핵심 기능이 될 것입니다.
