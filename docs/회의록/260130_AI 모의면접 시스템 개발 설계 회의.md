

## **26.01.30(금)🗒️ \[회의록\] AI 모의면접 시스템 개발 설계 회의**

**일시:** 2026년 1월 30일

**주제:** 시스템 아키텍처 설계 및 핵심 기능/데이터 구조 확립

**기술 스택:** Python, FastAPI, PostgreSQL, Redis

---

### **1\. 데이터베이스 및 인프라 설계 (DB Architecture)**

#### **1.1 PostgreSQL (영구 저장소)**

사용자의 기본 정보와 면접 결과 등 정형 데이터를 관리합니다.

* **User:** 사용자 프로필 (ID, PW, 인적사항, 직무, 희망기업, 이력서 ID 등)  
* **Resume:** 이력서 원본 및 파싱 데이터 (이력서 ID, PDF 추출 텍스트)  
* **Q\_A:** 면접 질문 및 답변 데이터 세트  
* **Result:** 면접 결과 리포트 (대화 로그, 타임라인별 평가, 장단점 및 보완점)  
* **Auth/Session:** 동시 접속 처리를 위한 세션 및 로그인 토큰 테이블

#### **1.2 Redis (인메모리 DB)**

실시간 면접 진행 시 빠른 응답 속도를 위해 사용합니다.

* **저장 항목:** 실시간 채팅 기록, 캐싱된 개인정보, 실시간 웹 서치 결과 등

#### **1.3 Backend Framework**

* **FastAPI:** 비동기(Async) 처리를 기본으로 하여 고성능 API 서버 구축

---

### **2\. 핵심 기능 정의 (Core Functions)**

| 구분 | 주요 내용 |
| :---- | :---- |
| **기본 프로세스** | 질문 생성 → 사용자 답변 → AI 실시간 평가 루프 |
| **데이터 전처리** | PDF-to-Text (이력서 분석), STT (음성 인식), TTS (AI 음성 출력) |
| **비언어적 분석** | **Sound-to-Text:** 음성의 높낮이, 떨림 등 분석 **Vision-to-Text:** 표정 및 자세 분석을 통한 태도 점수화 |
| **인터페이스** | 웹 기반 UI 구현 및 AI 아바타(Face 생성) 연동 검토 |

---

### **3\. 면접 고도화 및 페르소나 설정**

* **면접관 페르소나:**  
  * 사용자가 직접 선택하거나 이력서 정보를 기반으로 자동 생성.  
  * **유형 세분화:** 꼰대형, 친절형, 압박 면접형 등.  
  * 페르소나에 따른 맞춤형 **꼬리질문** 생성 로직 구현.  
* **피드백 시스템:**  
  * 단순 점수 부여를 넘어 **이력서 수정 제안** (프로젝트 경험 및 논리적 보완점) 포함.

---

### **4\. 실시간성 및 확장성 전략 (Advanced Logic)**

#### **4.1 동시성 관리**

* 단일 서버에서 다수 사용자가 동시에 면접을 볼 수 있도록 세션 기반 관리 및 비동기 프로그래밍 적극 활용.

#### **4.2 실시간 멀티모달 처리**

* **멀티모달 입출력:** 텍스트, 음성, 영상 데이터의 동시 처리.  
* **끼어들기(Interruption):** 사용자가 답변 중일 때 AI가 개입하거나 흐름을 끊는 실시간 반응성 구현.

#### **4.3 구현 방법론 (Modularization)**

* 기본적인 \*\*정적 테스트 모듈(Core)\*\*은 유지하되, 실시간 기능이나 비언어적 분석 기능은 \*\*데코레이터(Decorator)\*\*나 **함수형 모듈** 형태로 추가하여 코드 유지보수성 향상.

---

### **5\. 향후 과제 및 보완 사항**

* **On-Premise 전환:** 초기 API 모델 사용 후, 보안 및 비용 절감을 위해 로컬 저장형 모델(Llama 3 등) 도입 검토.  
* **데이터셋 정교화:** 질문/답변 카테고리(기술, 인성) 및 평가 항목(직무 이해도, 협업 등)의 세부 설계.  
* **OCR 고도화:** 신분증 사진 촬영 등을 통한 사용자 정보 자동 입력 및 수정 기능.
