# AI 모의면접 결과 페이지 구현 – 중간 진행 기록

## 1. 작업 개요
본 기록은 AI 모의면접 프로젝트에서 **면접 결과 페이지(Result Page)** 를 구현·개선하는 과정에서의  
기술적 진행 상황과 시행착오를 함께 정리한 문서이다.

현재 단계에서는 **세션 단위 결과 조회 기능까지는 완성되지 않았으며**,  
단일 세션 결과를 기준으로 한 결과 페이지 UI 및 데이터 연동 검증에 초점을 두었다.

---

## 2. 작업 목표
- FastAPI 백엔드에서 반환되는 면접 결과 데이터를 기반으로 결과 페이지 렌더링
- 점수, 합격 여부(PASS/FAIL), 요약 피드백 시각화
- 레이더(스파이더) 차트를 활용한 역량 프로파일 시각화
- 라이트 / 다크 모드 환경 모두에서 가독성 확보
- 결과 페이지 단독 실행 및 검증 가능 상태 확보

---

## 3. 주요 구현 내용

### 3.1 결과 페이지 기본 구성
- 총점(score)
- 최종 결과 배지(PASS / FAIL)
- 면접 요약(summary)
- 생성 시각(created_at)
- 역량별 피드백 카드
- 근거 보기 / 접기 토글 기능

프론트엔드는 **정적 result.html + JS(fetch)** 구조로 구현하여  
백엔드 API(`/report/session/{session_id}/result`)와 직접 통신하도록 구성하였다.

---

### 3.2 레이더 차트(역량 프로파일) 구현
- Chart.js 기반 radar chart 사용
- 기술 역량 / 문제 해결 / 커뮤니케이션 / 태도 항목 시각화
- 점수 범위: 1~5

#### 라이트 / 다크 모드 대응
- body `data-theme` 값을 기준으로 차트 색상 분기
- 항목 라벨, 눈금, 격자선 색상을 모드별로 다르게 설정
- 다크 모드에서는 가독성 양호
- 라이트 모드에서는 색상 대비 문제를 발견하고 수정

---

### 3.3 UI / UX 개선 작업
- 카드 형태 UI 통일
- 점수 카드 강조(숫자 크기, 배지 강조)
- 피드백 카드 내 `근거 보기` 버튼 추가
- 버튼 클릭 시 근거 영역 토글 표시
- 화면 크기 변경 시 레이아웃 깨짐 최소화 (grid + media query)

---

## 4. 시행착오 및 문제 해결 기록

### 4.1 CSS 구조 누적 문제
- 디자인 개선 과정에서 CSS를 “추가” 위주로 진행
- 동일 selector 중복 선언(`body`, `:root`)이 누적됨
- 결과적으로:
  - 배경 그라데이션 효과 상실
  - 하단 콘텐츠 잘림처럼 보이는 현상 발생
  - 페이드 효과가 카드 내용을 가리는 문제 발생

👉 **정리 없이 누적된 스타일이 문제의 핵심 원인이었음**

---

### 4.2 하단 잘림 현상
- 배경 고정 + 하단 페이드 효과 적용 중
- `position: fixed` 사용으로 콘텐츠 위를 덮는 문제 발생
- 근거 보기 펼쳤을 때 카드 하단이 가려지는 현상 발생

👉 이후:
- `fixed` → `absolute` 구조로 수정
- `.wrap` 하단 padding 보정
- 카드 overflow 설정 점검

완전한 해결보다는 **시각적 완화 상태**까지 도달

---

### 4.3 환경 문제 (가장 큰 혼란 요인)
#### 증상
- 프론트엔드에서 API 호출 실패
- 브라우저 콘솔: `Failed to fetch`
- 서버 로그: `500 Internal Server Error`
- psycopg2 `connection refused` 오류

#### 원인
- **PostgreSQL Docker 컨테이너 미실행 상태**
- 백엔드는 정상 실행 중이었으나 DB 연결 불가

#### 해결
- Docker 컨테이너 실행 확인
- `docker ps`로 DB 컨테이너 상태 확인
- 이후 API 정상 응답 확인

👉 코드 문제가 아니라 **환경 문제였음**

---

## 5. 현재 상태 요약

### 가능한 것
- 결과 페이지 단독 실행
- 단일 세션 결과 조회 (session_id 직접 지정)
- 점수 / 요약 / 피드백 / 레이더 차트 정상 출력
- 라이트 / 다크 모드 전환 가능

### 아직 하지 않은 것
- 세션 목록 기반 결과 조회
- 결과 페이지 라우팅 구조 통합
- 결과 데이터 다중 세션 관리 UI

---

## 6. 느낀 점 및 회고
- UI 개선은 **추가보다 정리가 더 중요**하다는 것을 체감함
- 재부팅 + 환경 미실행(Docker) 하나로 큰 디버깅 시간이 소요됨
- “코드 문제”라고 단정하기 전에 **환경부터 확인해야 한다는 교훈**을 얻음
- 과거 개인 홈페이지 만들던 경험과 현재 작업이 겹쳐지며,
  여전히 UI/구조를 직접 만지는 작업에 흥미가 있음을 재확인함
