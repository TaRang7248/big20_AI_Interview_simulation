# 웹 기반 AI 면접 시뮬레이션 가이드북 (GUIDEBOOK)

## 1. 개요 (Overview)
본 프로젝트는 **웹 기반 AI 면접 시뮬레이션** 프로그램입니다. 사용자는 면접관(AI)과 상호작용하며 실제 면접과 유사한 환경에서 연습할 수 있습니다. 관리자는 면접 공고를 등록하고 지원자를 관리할 수 있으며, 지원자는 공고를 확인하고 생성일 및 마감일을 고려하여 면접에 응시할 수 있습니다.

### 주요 특징
- **회원가입 및 로그인**: 지원자와 관리자로 구분된 회원 시스템.
- **공고 관리**: 
    - 관리자: 채용 공고 생성, 수정, 삭제.
    - 지원자: 공고 목록 및 상세 정보(작성자, 생성일, 마감일, 내용) 확인.
- **면접 진행 (Simulation)**:
    - 웹캠 및 마이크 연동 테스트.
    - 단계별 면접 (자기소개, 역량 검증, 기술 면접 등).
    - STT(음성 인식) 및 TTS(음성 합성) 시뮬레이션.
- **결과 분석**: 면접 후 점수 및 합격 여부 예측 시각화.

---

## 2. 개발 환경 (Environment Setup)
이 프로젝트는 **Python (Flask)** 백엔드와 **Vanilla JS** 프론트엔드로 구성되어 있으며, **PostgreSQL** 데이터베이스를 사용합니다.

### 필수 요구 사항
- **OS**: Windows, macOS, Linux
- **Language**: Python 3.8+, JavaScript (ES6+)
- **Database**: PostgreSQL (Docker 컨테이너 권장, 기본 포트 5432)
- **Browser**: Chrome, Edge 등 모던 브라우저 (카메라/마이크 권한 필요)

### 설치된 라이브러리
- **Flask**: 웹 서버 프레임워크
- **psycopg2-binary**: PostgreSQL 데이터베이스 연동
- **python-dotenv**: 환경 변수 관리

---

## 3. 프로그램 실행 방법 (Execution)
터미널에서 `server.py`를 실행하면 서버가 시작됩니다.

```bash
# 1. 의존성 설치 (필요 시)
pip install flask psycopg2-binary python-dotenv

# 2. 서버 실행
python server.py
# 실행 시 브라우저가 자동으로 열리며 http://localhost:5000 으로 접속됩니다.
```

## 4. 주요 기능 사용법 (User Manual)

### 4.1 회원가입 및 로그인
1. 초기 화면에서 **회원가입** 링크를 클릭합니다.
2. 아이디, 비밀번호, 이름, 생년월일, 성별, 이메일, 주소, 전화번호를 입력합니다.
3. **회원 유형**을 선택합니다 (면접자 또는 관리자).
4. 가입 후 로그인합니다.

### 4.2 지원자 (Applicant)
1. **공고 확인**: 로그인 후 대시보드에서 '지원 가능한 공고' 목록을 확인합니다. 각 공고의 생성일과 마감일을 한눈에 볼 수 있습니다.
2. **공고 상세**: '확인하기' 버튼을 눌러 공고의 상세 정보 페이지로 이동합니다.
    - **생성일**: 공고가 등록된 날짜를 확인합니다.
    - **마감일**: 지원 마감일을 확인합니다.
    - **내용**: 직무 및 면접 관련 상세 내용을 확인합니다.
3. **면접 지원**: 상세 페이지 하단의 '지원하기' 버튼을 클릭합니다.
4. **환경 테스트**: 카메라와 마이크 상태를 확인합니다.
5. **면접 진행**: AI 면접관의 질문에 대답하며 진행합니다.
6. **결과 확인**: 면접 종료 후 점수 그래프와 합격 예측 결과를 확인합니다.
7. **내 정보 수정**: 비밀번호 확인 후 개인정보를 수정하거나 비밀번호를 변경할 수 있습니다.

### 4.3 관리자 (Admin)
1. **공고 관리**: 대시보드에서 채용 공고를 등록, 수정, 삭제할 수 있습니다.
2. **지원자 현황**: 면접에 응시한 지원자들의 점수와 결과를 모니터링할 수 있습니다.

## 5. 파일 구조 (File Structure)

```
/
├── server.py           # Flask 웹 서버 및 API 엔드포인트 정의
├── db/                 # 데이터베이스 관련 파일
├── index.html          # 메인 HTML 페이지 (SPA 구조, 공고 상세 템플릿 포함)
├── styles.css          # 전체 스타일링 (CSS)
├── app.js              # 프론트엔드 로직 (SPA 라우팅, API 호출, 공고 생성일 렌더링 등)
├── .env                # 환경 변수 설정 (DB 접속 정보 등)
└── GUIDEBOOK.md        # 프로젝트 가이드북 (본 파일)
```

### 주요 파일 설명
- **server.py**: PostgreSQL과 연동하여 REST API를 제공합니다. 공고 조회 시 `created_at` (생성일) 정보를 포함하여 반환합니다.
- **app.js**: 프론트엔드 로직을 담당합니다. API로부터 받은 공고 데이터를 가공하여 화면에 `생성일 | 마감일` 형태로 표시합니다.
- **index.html**: SPA 구조의 뷰 템플릿입니다. 공고 상세 페이지에 생성일(`created_at`)을 표시할 수 있는 DOM 요소가 추가되었습니다.

## 6. 데이터베이스 구조 (Database Schema)

### `users` 테이블
- `id_name` (PK): 사용자 아이디
- `pw`: 비밀번호
- `name`: 이름
- `type`: 사용자 유형 ('applicant', 'admin')
- 기타: `dob`, `gender`, `email`, `address`, `phone`

### `interview_announcement` 테이블
- `id` (PK): 공고 고유 번호 (Serial)
- `title`: 공고 제목
- `content`: 공고 내용
- `deadline`: 마감일
- `id_name`: 작성자 ID (FK -> users.id_name)
- `created_at`: 생성일 (기본값: 현재 시간)
