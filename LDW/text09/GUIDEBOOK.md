# AI 면접 시뮬레이션 가이드북 (AI Interview Simulation Guidebook)

본 가이드북은 웹 기반 AI 면접 시뮬레이션 프로그램의 개요, 환경 설정, 실행 방법 및 주요 기능에 대해 설명합니다.

---

## 1. 프로젝트 개요
이 프로그램은 지원자가 AI 면접관과 대화하며 면접 실전 감각을 익힐 수 있도록 돕는 웹 애플리케이션입니다. 지원자의 이력서를 분석하여 맞춤형 질문을 생성하고, 답변을 음성으로 녹음하면 STT(Speech-to-Text)를 통해 텍스트로 변환하여 실시간으로 평가 및 다음 질문을 이어갑니다.

---

## 2. 개발 및 실행 환경
- **운영체제**: Windows (권장)
- **언어**: Python 3.9+
- **데이터베이스**: PostgreSQL
- **프론트엔드**: HTML5, Vanilla CSS, JavaScript
- **백엔드**: FastAPI (Python)

---

## 3. 프로그램 실행 방법

### 3.1 사전 준비
- PostgreSQL 서버가 실행 중이어야 합니다.
- 프로젝트 루트 디렉토리에 `.env` 파일이 존재해야 하며 다음 정보가 설정되어 있어야 합니다:
  - `OPENAI_API_KEY`: OpenAI API 키
  - `POSTGRES_PASSWORD`: 데이터베이스 비밀번호
  - `DB_HOST`, `DB_NAME`, `DB_USER`, `DB_PORT`

### 3.2 서버 실행
터미널에서 아래 명령어를 실행합니다.
```bash
python server.py
```
서버가 정상적으로 실행되면 `http://127.0.0.1:8000`에서 접속 가능합니다.

---

## 4. 사용하는 모델과 라이브러리

### 4.1 AI 모델
- **GPT-4o (OpenAI)**: 면접 질문 생성, 답변 평가, 최종 면접 분석 및 합격 여부 판단.
- **Whisper-1 (OpenAI)**: 지원자의 음성 답변을 텍스트로 변환 (STT).

### 4.2 주요 라이브러리
- **FastAPI**: RESTful API 서버 구축.
- **psycopg2**: PostgreSQL 데이터베이스 연동.
- **pydantic**: 데이터 검증 및 스키마 정의.
- **pypdf**: 이력서(PDF) 텍스트 추출.
- **python-dotenv**: 환경 변수 관리.

## 5. 주요 기능 사용법

### 5.1 회원 관리
- **회원가입**: 면접자(Applicant) 또는 관리자(Admin)로 가입할 수 있습니다.
- **로그인/로그아웃**: 아이디와 비밀번호를 통해 접속합니다.
- **정보 수정**: 이메일, 주소, 전화번호 및 비밀번호를 변경할 수 있습니다.

### 5.2 면접 공고 관리 (관리자 전용)
- 새로운 면접 공고를 등록하고, 기존 공고를 수정하거나 삭제할 수 있습니다.

### 5.3 면접 진행 (면접자 전용)
1. **공고 선택**: 지원하고자 하는 공고의 '상세 정보' 페이지에서 '지원하기'를 클릭합니다.
2. **이력서 업로드/환경 테스트**: PDF 이력서를 업로드하고 카메라/마이크 상태를 확인합니다.
3. **면접 시작**: AI 면접관이 첫 번째 질문을 합니다.
4. **답변 및 제출**: 음성으로 답변한 뒤 '답변 제출' 버튼을 클릭합니다. 총 12개의 질문으로 구성됩니다.
5. **세션 그룹화**: 각 면접은 '면접-1', '면접-2'와 같이 그룹화되어 DB에 저장됩니다.

### 5.4 면접 결과 확인
- 면접이 종료되면 AI가 전체 답변을 분석하여 기술, 문제해결, 의사소통, 태도 점수를 매기고 합격/불합격을 결정합니다. 관리자는 '지원자 현황' 메뉴에서 상세 기록을 볼 수 있습니다.

## 6. 파일 구조 설명
- `server.py`: FastAPI를 이용한 백엔드 API 및 비즈니스 로직.
- `index.html`: 단일 페이지 애플리케이션(SPA) 구조의 프론트엔드 메인 페이지.
- `app.js`: 프론트엔드 동적 로직 및 API 통신.
- `styles.css`: 디자인 및 레이아웃 스타일링.
- `uploads/`: 업로드된 이력서 및 음성 답변 파일 저장 경로.
- `data.json`: 초기 질문 데이터 풀 (사용자 정의 가능).
- `requirements.txt`: 프로젝트 의존성 목록.
- `diagnose_db.py`, `create_table.py`: DB 설정 및 진단 도구.
