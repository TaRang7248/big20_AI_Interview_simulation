# AI 면접 시뮬레이션 가이드북 (GUIDEBOOK)

## 1. 프로젝트 개요 (Overview)
본 프로젝트는 **웹 기반 AI 면접 시뮬레이션 프로그램**입니다.
사용자는 가상의 면접관(AI)과 함께 화상 면접을 진행하며, 면접 과정을 통해 자신의 역량을 점검하고 피드백을 받을 수 있습니다.
관리자는 면접 공고를 등록하고 지원자 현황을 관리할 수 있습니다.

---

## 2. 개발 환경 및 요구사항 (Environment)
본 프로그램은 다음 환경에서 개발 및 테스트되었습니다.
*   **OS**: Windows
*   **Language**: Python 3.x, JavaScript (ES6+)
*   **Database**: PostgreSQL
*   **Backend Framework**: Flask
*   **Frontend**: Vanilla HTML/CSS/JS

### 필수 라이브러리 (Dependencies)
`requirements.txt`에 명시된 주요 라이브러리는 다음과 같습니다.
*   `Flask`: 웹 서버 프레임워크
*   `psycopg2` / `psycopg2-binary`: PostgreSQL 데이터베이스 연동
*   `python-dotenv`: 환경 변수 관리

---

## 3. 프로그램 실행 방법 (How to Run)
1.  **데이터베이스 설정**:
    *   PostgreSQL이 설치되어 있어야 하며, `interview_db` 데이터베이스가 생성되어 있어야 합니다.
    *   `.env` 파일에 데이터베이스 접속 정보(`POSTGRES_PASSWORD` 등)를 설정합니다.

2.  **서버 실행**:
    터미널에서 프로젝트 루트 디렉토리(`server.py`가 있는 위치)로 이동 후 다음 명령어를 실행합니다.
    ```bash
    python server.py
    ```

3.  **웹 브라우저 접속**:
    서버가 정상적으로 실행되면 브라우저가 자동으로 열리거나, 수동으로 `http://localhost:5000`에 접속합니다.

## 4. 주요 기능 사용법 (Features)
### 4.1. 회원가입 및 로그인
*   **회원가입**: 아이디, 비밀번호, 이름, 생년월일, 성별, 이메일, 주소, 전화번호를 입력하여 가입합니다. '면접자' 또는 '관리자' 유형을 선택할 수 있습니다.
*   **로그인**: 가입한 아이디와 비밀번호로 로그인합니다.

### 4.2. (면접자) 면접 진행
*   **공고 지원**: 대시보드에서 등록된 공고를 확인하고 '지원하기'를 클릭합니다.
*   **환경 점검**: 카메라와 마이크 상태를 테스트합니다.
*   **면접 실행**: 가상의 AI 면접관 질문에 따라 답변을 진행합니다. (Phase 1: 자기소개, Phase 2: 역량 검증, Phase 3: 기술/코딩)
*   **결과 확인**: 면접 종료 후 예상 점수와 합격/불합격 여부를 확인합니다.

### 4.3. (면접자) 내 정보 관리
*   **정보 수정**: 비밀번호 확인 후 이메일, 주소, 전화번호를 수정할 수 있습니다.
*   **비밀번호 변경**: 현재 비밀번호 확인 후 새로운 비밀번호로 변경할 수 있습니다.

### 4.4. (관리자) 공고 관리
*   **공고 등록**: 공고 제목, 마감일, 내용을 입력하여 새 공고를 등록합니다. **작성자 ID**가 자동으로 기록됩니다.
*   **공고 수정/삭제**: 등록된 공고를 수정하거나 삭제할 수 있습니다.
*   **지원자 현황**: 지원자들의 면접 결과 및 통계를 확인할 수 있습니다.

## 5. 파일 구조 설명 (File Structure)
```
/
├── server.py           # 메인 백엔드 서버 (Flask)
├── index.html          # 메인 프론트엔드 페이지 (SPA 구조)
├── styles.css          # 스타일시트
├── app.js              # 프론트엔드 로직 (라우팅, API 호출, 상태 관리)

├── create_table.py     # (유틸리티) 테이블 생성 스크립트
├── check_table.py      # (유틸리티) 테이블 확인 스크립트
├── migrate_db.py       # (유틸리티) DB 마이그레이션 스크립트
├── test_job_api.py     # (유틸리티) API 테스트 스크립트
└── db/                 # (구) SQLite 데이터 파일 저장소 (현재는 PostgreSQL 사용)
```

## 6. 문제 해결 (Troubleshooting)
*   **포트 충돌(Address already in use)**: 이미 `server.py`가 실행 중인지 확인하고 종료 후 다시 실행하세요.
*   **DB 연결 오류**: PostgreSQL 서비스가 실행 중인지, `.env` 파일의 비밀번호가 정확한지 확인하세요.
